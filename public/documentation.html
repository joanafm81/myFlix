Read me

myFlix is a web applicattion with information about different movies, directors, and genres. Users can acess information, sign up, update their personal details, and create a list of their favorite movies.

List of URL endpoints and request methods that correlate with each of the operations

1 Return a list of all movies to the user /movies GET None A JSON object holding data about all the movies

3 Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/[Genre]/[Name] GET None A JSON object holding data about a genre, containing a name, description. Example:
{
"Name": "Adventure",
"Description": "Genre description."
};
4 Return data about a director (bio, birth year, death year) by name /movies/[Director]/[Name] GET None A JSON object holding data about a director, containing a name, bio, birth year, death year. Example:

5 Allow new users to register /users POST A JSON object holding data about the user to add, structured like:
{
"name": "Joana Martins",
"favoriteMovies": []
};
 A JSON object holding data about the user that was added, including an ID:
{
"id": "7656edb2-7437-4d0f-baa3-fa6ad60494ef",
"name": "Joana Martins",
"favoriteMovies": []
};
6 Allow users to update their user info (username) /users/[ID] PUT None A message indicating the username was updated
7 Allow users to add a movie to their list of favorites (showing only a text that a movie has been added) /users/[ID]/[Favorite%20Movie] POST A JSON object holding data about the user to add, structured like:
{
"name": "Joana Martins",
"favoriteMovies": []
};
 A message indicating the movie was updated
8 Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed) /users/[ID]/[Favorite%20Movie] DELETE None A message indicating the movie was removed
9 Allow existing users to deregister (showing only a text that a user has been removed) /users/[ID] DELETE None A message indicating the user was removed
